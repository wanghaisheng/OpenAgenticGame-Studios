# Publish Team

## 🎯 Overview

The Publish team contains specialized agents for game publishing, platform certification, store submission, and release management. These agents handle the critical final stages of game development where quality meets platform requirements and games reach players successfully.

## 🏗️ Team Structure

### **Publish Team Agents**
- **store-submission-specialist.md** - Store submission and platform compliance specialist
- **platform-certification-specialist.md** - Platform certification and compliance specialist
- **release-coordinator.md** - Release coordination and launch management specialist
- **publish-quality-gates.md** - Publishing quality gates and validation specialist
- **store-page-optimizer.md** - Store page optimization and marketing specialist
- **platform-relationship-manager.md** - Platform relationship and partnership specialist

### **Cross-Team Coordination**
- **Production Team**: `release-manager` - Release pipeline coordination
- **Quality Assurance Team**: `qa-lead` - Quality validation and testing
- **Support Team**: `community-manager` - Community and player support

## 🔧 Publishing Process

### **Phase 1: Preparation**
1. **Quality Validation**: Ensure game meets all quality standards
2. **Platform Requirements**: Verify platform-specific requirements
3. **Asset Preparation**: Prepare platform-specific assets
4. **Documentation**: Prepare submission documentation
5. **Compliance Check**: Final compliance validation

### **Phase 2: Submission**
1. **Platform Submission**: Submit to all target platforms
2. **Review Process**: Manage platform review process
3. **Issue Resolution**: Address submission issues and feedback
4. **Certification**: Complete platform certification process
5. **Approval**: Obtain final approval for release

### **Phase 3: Launch**
1. **Release Coordination**: Coordinate release across platforms
2. **Store Page Activation**: Activate store pages and listings
3. **Marketing Launch**: Coordinate marketing and promotion
4. **Community Engagement**: Engage with player community
5. **Monitoring Activation**: Activate monitoring and analytics

### **Phase 4: Post-Launch**
1. **Performance Monitoring**: Monitor launch performance
2. **User Feedback**: Collect and analyze user feedback
3. **Issue Resolution**: Address post-launch issues
4. **Updates**: Manage updates and patches
5. **Community Management**: Ongoing community engagement
