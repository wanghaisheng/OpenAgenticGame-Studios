# Context Management - Team-Based Architecture

## 🎯 Overview

Context management in the Claude Code Game Studios architecture ensures that all teams and agents have access to relevant information while maintaining focus and efficiency. This document outlines how context is managed across the 8-team structure.

## 🏗️ Team-Based Context Structure

### **Context Categories**
1. **Project Context**: Overall project information and goals
2. **Team Context**: Team-specific information and responsibilities
3. **Agent Context**: Individual agent knowledge and expertise
4. **Skill Context**: Specialized skill knowledge and capabilities
5. **Engine Context**: Engine-specific information and best practices

### **Strategic Context**

- Technical strategy and architecture

### **Decision Context**

## 🔄 Context Sharing

### **Cross-Team Context Sharing**
- **Leadership Team**: Shares strategic context with all teams
- **Programming Team**: Shares technical context with relevant teams
- **Design Team**: Shares design context with implementation teams
- **Art Team**: Shares visual context with implementation teams
- **QA Team**: Shares quality context with all teams
- **Production Team**: Shares operational context with all teams

### **Context Updates**
- **Real-time Updates**: Critical information shared immediately
- **Scheduled Updates**: Regular context synchronization
- **Event-Driven Updates**: Context updates based on events
- **On-Demand Updates**: Context available when needed

## 📊 Context Metrics

### **Context Quality**
- **Completeness**: All relevant context available
- **Accuracy**: Context information is accurate and up-to-date
- **Relevance**: Context is relevant to current work
- **Timeliness**: Context is current and timely

### **Context Usage**
- **Access Frequency**: How often context is accessed
- **Usage Patterns**: How context is used by teams
- **Effectiveness**: How well context supports work
- **Efficiency**: How efficiently context is retrieved
