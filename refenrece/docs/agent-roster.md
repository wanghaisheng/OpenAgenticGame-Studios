# Agent Roster - Team-Based Organization

The following agents are available, organized by team structure. Each has a dedicated definition file in `refenrece/agents/[team]/`. Use the agent best suited to the task at hand. When a task spans multiple domains, the coordinating agent (usually `producer` or the domain lead) should delegate to specialists.

## 🎯 Team Structure Overview

- **Leadership Team** (7 agents) - Project vision and strategic direction
- **Programming Team** (28 agents) - Technical implementation and engine expertise  
- **Design Team** (12 agents) - Game design and content creation
- **Art Team** (10 agents) - Visual art and user experience
- **Quality Assurance Team** (6 agents) - Testing and quality control
- **Production Team** (12 agents) - Project management and operations
- **Support Team** (2 agents) - Advanced technical support
- **Enhanced Team** (4 agents) - Enhanced capabilities

## 🔄 Usage Guidelines

### **Task Assignment**
1. **Start with team lead**: Begin with the appropriate team lead for the domain
2. **Delegate to specialists**: Team leads delegate to specialists within their team
3. **Cross-team coordination**: For cross-domain tasks, use `producer` for coordination
4. **Enhanced capabilities**: Use enhanced agents for complex or advanced requirements

### **Model Selection**
- **Opus**: Leadership and strategic decisions
- **Sonnet**: Complex technical and design work
- **Haiku**: Simple, well-defined tasks and documentation

### **Team Collaboration**
- **Intra-team**: Team leads coordinate within their team
- **Inter-team**: Producer coordinates between teams
- **Escalation**: Issues escalate to team leads, then to leadership team
